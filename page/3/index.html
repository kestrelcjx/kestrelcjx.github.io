包：库单元

如果未设置源文件所属包，则会位于未命名包（默认包）中。
当编写一个Java源代码文件时，此文件通常被称为编译单元。
每个编译单元都必须有一个后缀.java，而在编译单元内则有一个public类，
该类的名称必须与文件的名称相同（包括大小写，但不包括后缀名.java）。
每个编译单元只能由一个pubic类（也可以没有）。
创建包时，按照惯例，package名称第一部分是类创建者的反顺序的Internet域名
（例如：com.kestrelcjx.xxx）。package名称将分解为机器上的一个目录。
Java解释器的运行过程如下：首先，找出环境变量CLASSPATH。CLASSPATH包含一个或
多个目录，用作.class文件的根目录。从根目录开始，解释器获取包的名称并将
每个句号替换为反斜杠，以从CLASSPATH根肿产生一个路径名称。得到的路径会与CLASSPATH路径
中的各个不同项相连接，解释器就在这些目录中查找与你所创建类的名称相关的.class文件。

初始化与清理

用构造器确保初始化

方法重载

  • 区分重载方法：重载方法名称相同，参数数量、类型或顺序不同，但无法以返回值不同实现重载。
  • 涉及基本类型的方法重载：基本类型从一个“较小”的类型自动提升至一个“较大”的类型。
    char略有不同，如果没有char参数的方法，将直接提升至int。

默认构造器

如果不提供构造器，将自动创建一个无参数的默认构造器，如果提供了构造器，则不会自动创建。

this关键字

  • 使用对象调用方法，编译器会暗自将“所操作对象的引用”作为第一个参数传递给方法。
  • 使用this关键字在方法内部获得当前对象的引用，但只能在方法内部使用。
  • 在构造器中可以使用this()调用重载的其他构造器。
  • static方法中，不能使用this，且不能调用非静态方法。

清理：终结处理和垃圾回收

Java中的对象创建在堆中，对象不再使用时必须有垃圾回收器回收所占用内存。
发生垃圾回收时，会使用对象的finalize()方法（Object类中的protected方法）。
垃圾回收一边回收空间，一边使堆中的对象紧凑排列。

  • 对象可能不被垃圾回收
  • 垃圾回收并不等于析构
  • 垃圾回收只与内存有关

垃圾回收机制：

  1. 引用记数是一种简单但速度很慢的垃圾回收技术。每个对象都含有一个引用记数器，
    当有引用连接至对象时，引用记数加1。当离开作用域或被设置为null时，引用记数减1。
    垃圾回收器会在含有全部对象的列表上遍历，当发现某个对象引用记数为0时，就释放
    其所占用的空间（但是，引用记数模式通常会在记数值为0时立即释放对象）。引用记数
    常用来说明垃圾收集的工作方式，但似乎从未被应用于如何一种Java虚拟机实现中。
  2. 另一种垃圾回收：从堆栈和静态存储区开始，遍历所有的引用，找到所有“活”的对象
    。对于发现的每个引用，必须追踪它所引用的对象，然后是此对象包含的所有引用，直到
    “根源于堆栈和静态存储区的引用”所形成得到网络全部被访问为止。在这种方式下，Java虚拟机
    将采用一种自适应的垃圾回收技术，至于如何处理找到的存活对象，取决于不同的Java虚拟机实现。
    （例如：“停止-复制”、“标记-清扫”）

构造器初始化

  • 在类的内部，变量定义的先后顺序决定了初始化的顺序。即使变量定义散步于方法定义之间，
    它们仍旧会在方法（包括构造器）被调用之前得到初始化。
  • 静态初始化只有在必要时刻才会进行。如果不创建对象，也不引用对象的静态变量，那么不会进行初始化。
    初始化顺序是先静态对象，而后是非静态对象。
  • 显示的静态初始化（静态块）
  • 非静态实例化初始化
    与静态初始化子句相似，但少了static关键字，它保证无论调用哪个显示构造器，都会执行其中的初始化。

总结一下对象的创建过程，假设有一个名为Dog的类：

  1. 即使没有显示地使用static关键字，构造器实际上也是静态方法。因此，当首次创建类型为Dog
    的对象时（构造器可以看成是静态方法），或者Dog类静态方法/静态域首次被访问时，Java解释器必须查找类路径，
    以定位Dog.class文件。
  2. 然后载入Dog.class，有关静态初始化的所有动作都会被执行。因此，静态初始化只在Class对象

  4. 这块存储空间会被清零，这就自动地将Dog对象中的所有基本类型数据都设置为默认值，而引用类型为null。
  5. 执行所有出现于字段处的初始化动作。
  6. 执行构造器。

虚树

2019-08-27

虚树

用途：在树上做DP时，有时候我们并不需要从某些点进行转移，只需要从某些关键点便可求得答案，多余的点会增加算法的时间复杂度。虚树就是在原来的树基础上去掉多余点、只保留关键点，关键点包括指定的点以及他们的最近公共祖先(LCA)。

实现

首先，将某一非指定点作为虚树的根，将其放入栈中；然后，求出树上点的dfs序，将指定的点按dfs序从小到大排序后，依次放入栈中。在将点放入栈时，分情况讨论：
1.如果栈中元素个数小于等于1个，直接入栈，返回；否则求出栈顶元素与入栈元素的LCA。
2.如果栈顶元素为LCA，直接返回，元素不需要入栈；如果栈顶元素不为LCA，循环判断栈中元素个数大于1且第二个元素dfs序不小于LCA的dfs序，将栈顶元素和栈中第二个元素建边并将栈顶元素出栈，否则跳出循环；
3.如果在循环结束，栈顶元素不为LCA，将栈顶元素与LCA建边，栈顶元素出栈，再将LCA入栈；
4.将指定点入栈。

斯坦纳树(Steiner Tree)

2019-08-26

斯坦纳树(Steiner Tree)

用途：在图中求包含特定点的生成树，可以包含额外的点，最小生成树属于最小斯坦纳的一种。
算法复杂度O($n*3^k$)，n为图顶点个数，k为需要包含的特殊点个数。

求解最小斯坦纳树

求解最小斯坦纳树使用动态规划思想，用dp[S][u]表示以u为根节点、状态S的最小花费，S的二进制各位表示是否包含各点。
转移方式有两种：
1.将S分为两部分:dp[S][u] = min(dp[S][u], dp[S0][u]+dp[S^S0][u])通过枚举各种转移状态即可。
2.通过边u-v转移:dp[S][u] = min(dp[S][u], dp[S][v]+w[u][v])通过最短路算法进行转移。

快速沃尔什变换(FWT)

2019-06-08

快速沃尔什变换(FWT)

用途：求
$$C(i) = \sum_{i=j@k}A(j)*B(k)$$
其中@为位运算或(|), 与(&), 异或(^)中一种,算法复杂度O($n\log(n)$)。

1. 或运算

令 $$FWT[A] = (A_1, A_2…)$$
其中
$$A_i = \sum_{i=i|j}A[j]$$
用A0表示A的前一半，A1表示A的后一半，那么如果已经求得FWT[A0]和FWT[A1]，则可以得到
FWT[A] = (FWT[A0], FWT[A0]+FWT[A1]) (‘+’ 对应位相加) 当A长度为1时，FWT[A] = A(A0最高
位为0，对FWT[A]后一半能贡献，而A1最高位为1，对FWT[A]前一半不能贡献)。所以，就可以
通过迭代分治，在O(nlogn)求得FWT[A]。
并且，由
$$C(i) = \sum_{i=j|k}A(j)*B(k)$$
得到FWT[C] = FWT[A]*FWT[B] (‘*’ 对应位相乘)，所以能在求得FWT[A]和FWT[B]基础上，在
O(n)求得FWT[C]。
最后，需要将FWT[C]恢复为C。通过观察求FWT[A]的过程，发现只需要将FWT[C]的后半部分减
去FWT[C]的前半部分IFWT[C] = (IFWT[C0], IFWT[C1]-IFWT[C0]) 当C长度为1时，IFWT[C] = C
就可以得到C。所以，同样能通过逆向迭代分治，在O(nlogn)求得C。

2. 与运算

FWT的与运算和或运算非常类似，只需改变FWT[A]中元素含义，让FWT[A]中
$$A_i = \sum_{i=i\&j}A[j]$$
同样我们能发现FWT[A] = (FWT[A0]+FWT[A1], FWT[A1]),同时FWT[C] = FWT[A]*FWT[B]
所以同样能求得FWT[C],最后由IFWT[C] = (IFWT[C0]-IFWT[C1], IFWT[C1]),求得C

3. 异或运算

FWT的异或运算相比或运算和与运算有些不同，我们设i&j所得结果中1的个数的奇偶表示i&j的
奇偶性(偶：0；奇：1)，那么会有(i&k的奇偶性)^(j&k的奇偶性) = (i^j)&k的奇偶性，那么，令
$$A_i = \sum_{i\&k=0}A[k]-\sum_{i\&k=1}A[k]$$
其中i&k=0表示i&k为偶性，i&k=1表示i&k为奇性，我们能发现仍然有FWT[C] = FWT[A]*FWT[B]
假设不考虑二进制最高位已经求得FWT[A0]和FWT[A1],要求FWT[A]。那么再多考虑最高位，A0中
最高位为0，FWT[A0]仍然能直接贡献到FWT[A0]和FWT[A1]；而A1最高位为1，FWT[A1]能直
接贡献到FWT[A0]，但贡献到FWT[A1]时，因为有了最高位1，i&k的奇偶性改变。所以，有
FWT[A] = (FWT[A0]+FWT[A1], FWT[A0]-FWT[A1])最后求得FWT[C]后，由
IFWT[C] = ((IFWT[C0]+IFWT[C1])/2, (IFWT[C0]-IFWT[C1])/2),恢复得到C

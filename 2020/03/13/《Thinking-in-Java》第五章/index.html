初始化与清理

用构造器确保初始化

方法重载

  • 区分重载方法：重载方法名称相同，参数数量、类型或顺序不同，但无法以返回值不同实现重载。
  • 涉及基本类型的方法重载：基本类型从一个“较小”的类型自动提升至一个“较大”的类型。
    char略有不同，如果没有char参数的方法，将直接提升至int。

默认构造器

如果不提供构造器，将自动创建一个无参数的默认构造器，如果提供了构造器，则不会自动创建。

this关键字

  • 使用对象调用方法，编译器会暗自将“所操作对象的引用”作为第一个参数传递给方法。
  • 使用this关键字在方法内部获得当前对象的引用，但只能在方法内部使用。
  • 在构造器中可以使用this()调用重载的其他构造器。
  • static方法中，不能使用this，且不能调用非静态方法。

清理：终结处理和垃圾回收

Java中的对象创建在堆中，对象不再使用时必须有垃圾回收器回收所占用内存。
发生垃圾回收时，会使用对象的finalize()方法（Object类中的protected方法）。
垃圾回收一边回收空间，一边使堆中的对象紧凑排列。

  • 对象可能不被垃圾回收
  • 垃圾回收并不等于析构
  • 垃圾回收只与内存有关

垃圾回收机制：

  1. 引用记数是一种简单但速度很慢的垃圾回收技术。每个对象都含有一个引用记数器，
    当有引用连接至对象时，引用记数加1。当离开作用域或被设置为null时，引用记数减1。
    垃圾回收器会在含有全部对象的列表上遍历，当发现某个对象引用记数为0时，就释放
    其所占用的空间（但是，引用记数模式通常会在记数值为0时立即释放对象）。引用记数
    常用来说明垃圾收集的工作方式，但似乎从未被应用于如何一种Java虚拟机实现中。
  2. 另一种垃圾回收：从堆栈和静态存储区开始，遍历所有的引用，找到所有“活”的对象
    。对于发现的每个引用，必须追踪它所引用的对象，然后是此对象包含的所有引用，直到
    “根源于堆栈和静态存储区的引用”所形成得到网络全部被访问为止。在这种方式下，Java虚拟机
    将采用一种自适应的垃圾回收技术，至于如何处理找到的存活对象，取决于不同的Java虚拟机实现。
    （例如：“停止-复制”、“标记-清扫”）

构造器初始化

  • 在类的内部，变量定义的先后顺序决定了初始化的顺序。即使变量定义散步于方法定义之间，
    它们仍旧会在方法（包括构造器）被调用之前得到初始化。
  • 静态初始化只有在必要时刻才会进行。如果不创建对象，也不引用对象的静态变量，那么不会进行初始化。
    初始化顺序是先静态对象，而后是非静态对象。
  • 显示的静态初始化（静态块）
  • 非静态实例化初始化
    与静态初始化子句相似，但少了static关键字，它保证无论调用哪个显示构造器，都会执行其中的初始化。

总结一下对象的创建过程，假设有一个名为Dog的类：

  1. 即使没有显示地使用static关键字，构造器实际上也是静态方法。因此，当首次创建类型为Dog
    的对象时（构造器可以看成是静态方法），或者Dog类静态方法/静态域首次被访问时，Java解释器必须查找类路径，
    以定位Dog.class文件。
  2. 然后载入Dog.class，有关静态初始化的所有动作都会被执行。因此，静态初始化只在Class对象
    首次加载时进行一次。
  3. 当用new Dog()创建对象时，首先将在堆上为Dog对象分配足够的存储空间。
  4. 这块存储空间会被清零，这就自动地将Dog对象中的所有基本类型数据都设置为默认值，而引用类型为null。
  5. 执行所有出现于字段处的初始化动作。
  6. 执行构造器。
